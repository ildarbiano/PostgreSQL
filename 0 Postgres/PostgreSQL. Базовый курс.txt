===================== Row-Level Security (RLS):
механизм контроля доступа к данным на уровне отдельных строк таблицы в PostgreSQL. Позволяет администраторам баз данных определять политики безопасности, ограничивающие доступ пользователей к определённым строкам в зависимости от контекста выполнения запроса и атрибутов самих строк. 
Цели: 			-обеспечить безопасность данных без изменения приложения, реализовать многопользовательские системы с разделением данных, соответствовать требованиям GDPR, HIPAA и другим стандартам. 
Когда использовать: 	-мультитенантные системы, персональные данные пользователей, системы с вертикальным разделением прав. 
Принцип работы:		-RLS основан на автоматической модификации запросов к таблицам с включённой защитой. Когда пользователь выполняет запрос к таблице с активированной политикой RLS, PostgreSQL автоматически добавляет условия фильтрации к запросу, сужая видимый набор данных только до тех строк, которые пользователь имеет право видеть. 


===================== OLAP (Online Analytical Processing):
термин, который означает «оперативная аналитическая обработка данных» или «анализ данных в реальном времени». 

===================== OLTP (Online Transaction Processing):
это транзакционные системы, основная область применения системы управления базами данных (СУБД) PostgreSQL. 	
Суть OLTP-систем:				быстрая обработка множества одновременных операций с гарантией целостности данных. Такие системы характерны для решений, требующих надёжной обработки транзакций, например финансовых систем, интернет-магазинов, систем бронирования. 
Некоторые характеристики OLTP: 			много коротких операций SELECT, INSERT, UPDATE, DELETE, точечная нагрузка, преимущественно по индексу, критичны отклик и масштабируемость. 
PostgreSQL 					обеспечивает работу в сценариях OLTP благодаря поддержке принципов ACID.
